# Google Apps Script セットアップ手順

## 1. Googleスプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 新しいスプレッドシートを作成
3. URLから **スプレッドシートID** をメモ  
   例: `https://docs.google.com/spreadsheets/d/【このID】/edit`

## 2. Apps Script を設定

1. スプレッドシートのメニュー → **拡張機能** → **Apps Script**
2. エディタが開いたら、デフォルトのコードを全削除
3. アプリの **設定画面** に表示されているスクリプトをコピー&ペースト
4. `YOUR_SPREADSHEET_ID` を手順1でメモしたIDに置き換える
5. **保存**（Ctrl+S）

## 3. ウェブアプリとしてデプロイ

1. 右上の **「デプロイ」** → **「新しいデプロイ」**
2. 種類: **ウェブアプリ**
3. 設定:
   - 説明: `鍵受注管理API`
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**（共有のため）
4. **「デプロイ」** をクリック
5. Googleアカウントの許可を求められたら **許可**
6. **ウェブアプリURL** をコピー

## 4. アプリに設定

1. アプリ右上の **歯車アイコン（⚙）** をクリック
2. コピーしたURLを貼り付け
3. **「接続テスト」** → 成功を確認
4. **「保存」**

## 5. チームで共有（GitHub Pages）

1. このコードをGitHubリポジトリにプッシュ
2. `npm run build` でビルド
3. GitHub Pages を `dist` フォルダで公開
4. URLをチームに共有

> 💡 全員が同じGASのURLを設定することで、スプレッドシートでデータを共有できます

## データの流れ

```
[アプリ A] ←→ [Google Apps Script] ←→ [Googleスプレッドシート]
[アプリ B] ←↗                          ↑
[アプリ C] ←↗                      全員で閲覧可能
```

## トラブルシューティング

**接続テストが失敗する場合：**
- GASのデプロイで「アクセスできるユーザー: 全員」になっているか確認
- デプロイ後に新しいURLが発行される場合があるので再コピー

**データが反映されない場合：**
- GASのスクリプトを再保存して再デプロイ
- 「↺ 取得」ボタンで最新データを取得
